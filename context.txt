================================================================================
PAYFLOW - PROJECT CONTEXT
Last updated: 2026-06-01
================================================================================

OVERVIEW
--------
PayFlow is a B2B multi-tenant payroll SaaS platform for Nigerian SMEs.
Built with Go backend and React TypeScript frontend.
Handles: employee management, salary structures, payroll processing with
Nigerian tax compliance (PAYE, pension, NHF, NSITF), multi-provider fund
disbursement, virtual wallets, leave management, loans, and billing.

TECH STACK
----------
Backend:  Go 1.26, Chi v5, GORM, PostgreSQL 16, Redis 7, Asynq
Frontend: React 19, TypeScript, Vite, Tailwind CSS v4, TanStack Query v5,
          Zustand, React Hook Form, Zod, React Router v7, Recharts,
          Framer Motion, Lucide React
Auth:     JWT (golang-jwt/v5), bcrypt passwords
Email:    SMTP via Hermes v2 templates, async queue via Asynq
PDF:      Maroto v2 (payslip generation)
Providers: KoraPay (virtual accounts, transfers), Paystack (transfers,
           billing, bank verification), VFD Bank (corporate accounts, transfers)

CURRENT MIGRATION VERSION: 000027 (payroll job tracking)

================================================================================
BACKGROUND JOBS
================================================================================

1. Payroll Payout Processing
   - Triggered after payroll approval
   - Processes transfers for each employee
   - Uses Asynq (Redis) or gocron (in-memory fallback)

2. Async Email Delivery
   - Invitation emails, password resets, payroll notifications
   - Queued via Asynq with retry logic
   - Falls back to synchronous if no Redis

3. Daily Reconciliation
   - Runs 1 min after startup, then every 24 hours
   - Verifies wallet balances match ledger totals

4. Weekly Provider Reconciliation
   - Runs 5 min after startup, then every 7 days
   - Cross-checks transfers with payment provider APIs

================================================================================
ROLES
================================================================================

super_admin  - Platform-wide access (all orgs, settings, reconciliation)
admin        - Full access to own business
operator     - Employee management, payroll creation, transfers, loans
approver     - View and approve/reject payroll runs
employee     - Self-service portal (profile, payslips, leave)
