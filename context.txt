## PROJECT OVERVIEW
PayFlow is a B2B payroll SaaS for Nigerian businesses. Go backend with Chi router,
GORM ORM, PostgreSQL, Redis. Deployed on Railway. Frontend on Netlify (React).

## CURRENT SCALING CAPACITY
~2,000 organizations (up from ~500 after Phase 2)

## ARCHITECTURE DECISIONS
- Stay monolith (team <10, domain boundaries evolving)
- Use golang-migrate for ALL migrations (not GORM AutoMigrate)
- PostgreSQL as primary datastore (ACID required for financial data)
- Redis for caching (cache-aside) + job queue (Asynq)
- Graceful degradation: app works without Redis (falls back to gocron, no cache)
- First microservice candidate: wallet service (when it becomes bottleneck)

### Paystack Provider + Enable/Disable (2026-05-28) ✅

Paystack added as third payment provider:
- internal/platform/paystack/ — client, provider, types (follows Korapay pattern)
- Implements TransferProvider + BulkTransferrer (100-item batches)
- Two-step flow: create recipient → initiate transfer
- HMAC-SHA512 webhook verification (x-paystack-signature)
- Webhook handler for transfer.success/failed/reversed events
- 13 tests (7 client + 6 provider) all passing

Enable/disable mechanism:
- ENABLED_PROVIDERS env var (comma-separated, e.g., "korapay,paystack")
- Double gate: provider must be enabled AND have API keys configured
- Default: "korapay,vfd" (backward compatible)
- Zero providers = warning log, transfers will fail gracefully

Files created:

- internal/platform/paystack/client_test.go, provider_test.go
- internal/api/handler/paystack_webhook_handler.go
